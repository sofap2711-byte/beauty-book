## 2. Технологические ограничения

### shadcn/ui — ВЕРСИЯ v4 на base-ui
- **НЕ использовать** примеры с Radix UI (`@radix-ui/react-dialog`) — они несовместимы.
- **НЕ использовать** `@radix-ui/react-*` пакеты — проект на `@base-ui/react`.
- `DialogTrigger` поддерживает `asChild`, но только через кастомный патч (см. `src/components/ui/dialog.tsx`). Если ломается — чинить в том же файле.

### Tailwind CSS — версия 3.4
- **НЕ использовать** синтаксис Tailwind v4 (`@theme`, `@import "tailwindcss"`).
- Анимации через `tailwindcss-animate` (уже установлен).
- Цвета настроены через CSS-переменные в `globals.css` и `tailwind.config.ts`.

### Next.js 14 App Router
- Все страницы в `src/app/`
- Динамические роуты: `[id]` (не `[[...slug]]`)
- `next.config.mjs` → `images.unoptimized: true` (для статического экспорта если понадобится)

## 4. Работа с данными

### Все данные в `src/lib/data.ts`
- Услуги, мастера — статические константы
- Слоты — функция `getSlotsForDate(dateStr)` (детерминированный рандом)
- **Правило:** при добавлении новых услуг/мастеров обновлять `IconByName.tsx` (добавлять иконку в маппер)

### Нет сервера, нет API
- Форма записи не отправляет ничего никуда.
- Состояние формы — локальное (`useState` в `BookingDialog.tsx`).

## 5. Сложные компоненты — как чинить

### Dialog ломается / не открывается
Смотреть `src/components/ui/dialog.tsx`. Там кастомный патч для `asChild`:
```tsx
if (asChild && children) {
  return <DialogPrimitive.Trigger render={children as React.ReactElement} ... />
}
```

### Календарь — правки
- `src/components/Calendar.tsx` — полностью кастомный.
- Логика: `getDay()` возвращает 0 для воскресенья. В компоненте преобразуется к `Пн=0 ... Вс=6`.
- Навигация по месяцам через `useState(new Date())`.

### Слоты — логика занятости
- Функция `getSlotsForDate()` в `data.ts`.
- Если нужно изменить процент занятых слотов — править `r < 0.3` на другое число.
- Если нужно фиксированные занятые слоты — заменить рандом на хардкод.

## 6. Частые задачи — шпаргалка

### Добавить новую услугу
1. `src/lib/data.ts` → массив `services`
2. `src/components/IconByName.tsx` → добавить иконку в маппер
3. `src/lib/data.ts` → добавить мастеров в массив `masters` (указать `serviceId`)

### Добавить новую страницу
1. Создать `src/app/новый-путь/page.tsx`
2. Использовать `Header` и общие стили (см. существующие страницы)
3. Обновить `.kimi/CONTEXT.md` → раздел 6 и CHANGELOG

### Изменить цвета
1. `tailwind.config.ts` → `colors`
2. `src/app/globals.css` → CSS-переменные
3. Проверить все страницы на консистентность
