## 1. Общая концепция

Демо-сайт онлайн-записи для салона красоты.  
**Этап 1:** только визуал и путь пользователя, без базы данных. Все данные захардкожены.

Пользовательский путь:
1. Главная → выбирает услугу
2. Страница мастеров → выбирает мастера (или консультацию)
3. Календарь → выбирает дату
4. Слоты → выбирает время
5. Dialog с формой → подтверждает запись (демо-режим)

## 6. Страницы и компоненты (подробно)

### 6.1. Главная (`/`)
- **Header**: логотип (Scissors иконка + "BeautyBook"), кнопка "Войти" (декоративная)
- **Hero**: заголовок "Запишись к лучшим мастерам", описание, кнопка "Выбрать услугу" (скролл к секции)
- **Сетка услуг**: 3 карточки (Card) в колонку на мобильном, 3 в ряд на десктопе. Каждая: иконка, название, описание, цена, кнопка "Выбрать мастера" → `/service/[id]`

### 6.2. Мастера (`/service/[id]`)
- Заголовок — название услуги + цена
- Сетка мастеров: 1 колонка на мобильном, 2 на десктопе (`md:grid-cols-2`)
- **Карточка мастера:**
  - Avatar: круглый div 80×80 с первой буквой имени (`bg-slate-200`)
  - Имя + Instagram-иконка (ссылка на `instagram.com/{ник}`)
  - Должность + 5 звёздочек (Star иконки, `fill-amber-400`)
  - Кнопка "Консультация" → **Dialog** с двумя кнопками: MAX (чёрная) и Telegram (голубая)
  - Кнопка "Записаться" → `/master/{id}/book`

### 6.3. Календарь (`/master/[id]/book`)
- Заголовок "Выберите дату", подзаголовок с именем мастера
- **Calendar компонент** (кастомный, без библиотек):
  - Навигация по месяцам (стрелки)
  - Дни недели: Пн–Вс
  - Пн–Пт: белые кнопки, кликабельные
  - Сб–Вс: серые блоки, не кликабельны
  - Сегодня: обведён `ring-2 ring-sky-400`
  - Выбранный день: `bg-sky-400 text-white`
- При клике на дату → ниже появляется сетка слотов с анимацией `animate-fade-in-up`

### 6.4. Слоты
- Сетка: 3 колонки на мобильном, 4–5 на десктопе
- **Свободный:** `variant="outline"`, белый/голубой hover, кликабельный → открывает BookingDialog
- **Занятый:** серый фон `bg-slate-300`, текст зачёркнут, подпись "занято" мелким серым

### 6.5. BookingDialog (форма записи)
- Открывается при клике на свободный слот
- Показывает: мастер, дата, время
- Поля: Имя (обязательно), Телефон (обязательно, placeholder +7), Telegram (@ник), Комментарий (textarea)
- Кнопка "Подтвердить запись"
- При отправке:
  1. Показывается состояние "Запись успешна! (Демо-режим)" с галочкой
  2. Через 1.5 секунды Dialog закрывается
  3. Форма сбрасывается

## 7. Сложные / нестандартные моменты

### 7.1. shadcn/ui v4 на base-ui (не Radix)
- Классические примеры из интернета могут не работать — API другое.
- **Проблема:** `DialogTrigger` не поддерживал `asChild` из коробки.
- **Решение:** пропатчен `src/components/ui/dialog.tsx` — добавлена поддержка `asChild` через `render` prop base-ui.

### 7.2. Lucide-иконки по строке
- Нельзя динамически импортировать по имени в React без маппера.
- **Решение:** `IconByName.tsx` — явный маппер `name → компонент`.
- Доступные иконки: Sparkles, Heart, Flower2. Если добавлять новые услуги — расширять маппер.

### 7.3. Детерминированные слоты
- Нужно, чтобы при повторном открытии той же даты показывались те же занятые слоты.
- **Решение:** хэш-функция от строки даты, затем `Math.sin()`-генератор.

### 7.4. Tailwind + shadcn v4 конфликт
- Стандартный `globals.css` от shadcn v4 использовал `@import "tw-animate-css"` и `@import "shadcn/tailwind.css"`, которых не было в Tailwind v3.
- **Решение:** полностью переписан `globals.css` в классическом стиле shadcn v3 с HSL-переменными. Установлен `tailwindcss-animate`.

### 7.5. Календарь без библиотеки
- Реализован вручную на CSS Grid.
- Учитывает смещение первого дня месяца (понедельник = 0).
- Поддержка навигации по месяцам.
